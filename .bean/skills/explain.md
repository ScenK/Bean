# Explain

Explain the provided code, file, or page so a competent newcomer gets it fast:

- Start with the one-line "what is this and why does it exist".
- Walk the main flow in order, naming the key pieces and how they connect.
- Point out the non-obvious bits — the gotchas someone would trip on.
